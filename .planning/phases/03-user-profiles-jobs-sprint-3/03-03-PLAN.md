# Phase 3, Plan 3: Job Search & Frontends (HIQ-026, 030)

**Target Phase:** 03-user-profiles-jobs-sprint-3
**Goal:** Build paginated job list `GET /jobs` with PostgreSQL full-text search, skills array containment, and Redis caching. Build the Job Board UI in Next.js.

## Technical Context
- **Dependencies:** 03-02-PLAN.md (Job model and TSVECTOR search_vector)
- **Key Constraints:** `GET /jobs` must cache results in Redis, invalidated by the `jobs:version` key. UI must implement React Hook Form for filters.
- **New files:** `backend/app/api/v1/jobs.py` (append `GET /`), `frontend/app/jobs/page.tsx`, `frontend/components/jobs/JobCard.tsx`

## Action Plan

### 1. Job Search Endpoint (`backend/app/api/v1/jobs.py`)
- [ ] Add `GET /` to `jobs_router`:
  - Query parameters: `q: str`, `skills: List[str]`, `location: str`, `min_salary: int`, `page: int = 1`, `limit: int = 10`.
  - Calculate cache key hash based on params + `jobs:version`.
  - If in cache, return.
  - Build SQLAlchemy query:
    - Base: `select(Job).where(Job.is_deleted == False, Job.is_active == True)`
    - If `q`: `.where(Job.search_vector.bool_op("@@")(func.websearch_to_tsquery("english", q)))`
    - If `skills`: `.where(Job.skills.contains(skills))`
    - If `location`: `.where(Job.location.ilike(f"%{location}%"))`
    - If `min_salary`: `.where(Job.salary_min >= min_salary)`
  - Apply `offset` and `limit`. Execute query.
  - Fetch total count for pagination metadata.
  - Cache result in Redis for 5 minutes. Return JSON.

### 2. Job Board UI (`frontend/app/jobs/page.tsx`)
- [ ] Create `JobsPage` server/client component:
  - Sidebar with filters (Search input, Location input, Minimum Salary slider, Skills multiselect).
  - Main area showing paginated job cards.
- [ ] Implement `JobCard` component (`frontend/components/jobs/JobCard.tsx`) showing title, location, salary range, and skills tags.
- [ ] Connect filters to URL query parameters or Zustand store.
- [ ] Fetch data via `api.ts` to the backend `GET /v1/jobs`.

## Verification
- [ ] `GET /v1/jobs?q=engineer` returns matching results.
- [ ] Subsequent identical `GET` requests return instantly (cache hit).
- [ ] Mutating a job (create/update) breaks the cache version, and the next `GET` fetches fresh data.
- [ ] Frontend job board correctly renders search results and pagination controls.
