# Phase 3, Plan 2: Job Posting CRUD (HIQ-024, 025, 027, 028, 029)

**Target Phase:** 03-user-profiles-jobs-sprint-3
**Goal:** Design the Job database model with TSVECTOR, and implement the create, edit, update, delete (soft delete), and close job endpoints.

## Technical Context
- **Dependencies:** 02-02-PLAN.md (RoleChecker `require_recruiter`)
- **Key Constraints:** `Job.search_vector` generated column for FTS. Soft delete pattern using `is_deleted = True`. Redis cache invalidation on mutations.
- **New files:** `backend/app/models/job.py`, `backend/app/schemas/job.py`, `backend/app/api/v1/jobs.py`
- **Modify:** `backend/app/main.py` (register router)

## Action Plan

### 1. Database Model (`backend/app/models/job.py`)
- [ ] Create `Job` SQLAlchemy ORM model.
  - Columns: `id`, `title`, `description`, `salary_min`, `salary_max`, `location`, `is_active`, `is_deleted`, `recruiter_id`, `created_at`, `updated_at`.
  - Column `skills` of type `ARRAY(String)`.
  - Column `search_vector` of type `TSVECTOR` with `Computed("to_tsvector('english', coalesce(title, '') || ' ' || coalesce(description, ''))", persisted=True)`.
- [ ] Add GIN index on `search_vector`.
- [ ] Add relationship to `User` model.

### 2. Schemas (`backend/app/schemas/job.py`)
- [ ] Create `JobCreate(BaseModel)` and `JobUpdate(BaseModel)`.
- [ ] Create `JobResponse(BaseModel)` including all public fields.

### 3. API Endpoints (`backend/app/api/v1/jobs.py`)
- [ ] Create router with `prefix="/jobs"`, `tags=["Jobs"]`.
- [ ] `POST /`:
  - Enforce `require_recruiter`. Create job record. Increment `jobs:version` in Redis.
- [ ] `GET /{job_id}`:
  - Fetch job by ID. If `is_deleted`, return 404.
- [ ] `PUT /{job_id}`:
  - Enforce `require_recruiter` and ownership check. Update job. Increment `jobs:version`.
- [ ] `DELETE /{job_id}`:
  - Enforce `require_recruiter` and ownership check. Set `is_deleted = True`. Increment `jobs:version`.
- [ ] `PATCH /{job_id}/close`:
  - Enforce `require_recruiter` and ownership check. Set `is_active = False`. Increment `jobs:version`.

### 4. Integration
- [ ] Register `jobs_router` in `backend/app/main.py`.

## Verification
- [ ] `POST /v1/jobs` as recruiter succeeds; as candidate fails (403).
- [ ] `PUT /v1/jobs/{id}` updates job and increments Redis `jobs:version`.
- [ ] `DELETE /v1/jobs/{id}` soft deletes job; subsequent `GET /{id}` returns 404.
