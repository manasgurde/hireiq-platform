# Phase 4, Plan 1: Resume Uploads (HIQ-031 to HIQ-034)

**Target Phase:** 04-resume-applications-sprint-4
**Goal:** Setup S3 bucket integrations, implement `POST /resumes/upload-url` (presigned), `POST /resumes/confirm`, `GET/DELETE /resumes/me`, and PyMuPDF text extractor.

## Technical Context
- **Dependencies:** 03-RESEARCH.md (S3 config via `aioboto3`), 04-RESEARCH.md (Presigned URLs + PyMuPDF)
- **Key Constraints:** Resume parsing must use `anyio.to_thread.run_sync` to avoid blocking the event loop.
- **New files:** `backend/app/api/v1/resumes.py`, `backend/app/schemas/resume.py`, `backend/app/models/resume.py`, `backend/app/core/pdf_parser.py`
- **Modify:** `backend/app/main.py`, `backend/app/core/s3.py`

## Action Plan

### 1. Database Model (`backend/app/models/resume.py`)
- [ ] Create `Resume` SQLAlchemy model.
  - Columns: `id` (UUID), `candidate_id` (FK to `users.id`, unique), `s3_url`, `raw_text` (Text), `created_at`, `updated_at`.
  - Relationship to `User`.

### 2. S3 Core Update (`backend/app/core/s3.py`)
- [ ] Add `async def generate_presigned_upload_url(object_name: str, content_type: str = "application/pdf") -> str`.
  - Use `aioboto3` to generate a `put_object` URL valid for 3600 seconds.
- [ ] Add `async def get_s3_object_bytes(object_name: str) -> bytes`.

### 3. PDF Parser (`backend/app/core/pdf_parser.py`)
- [ ] Implement `extract_pdf_text_from_bytes_sync(pdf_bytes: bytes) -> str` using `fitz.open(stream=pdf_bytes, filetype="pdf")`.
- [ ] Implement async wrapper `async def parse_resume_text(pdf_bytes: bytes) -> str` utilizing `anyio.to_thread.run_sync`.

### 4. API Endpoints (`backend/app/api/v1/resumes.py`)
- [ ] `POST /upload-url`:
  - Require candidate role.
  - Generate a secure S3 key: `resumes/{user_id}/{uuid4()}.pdf`.
  - Return `{ "upload_url": presigned_url, "s3_key": s3_key }`.
- [ ] `POST /confirm`:
  - Accept `s3_key`.
  - Fetch the file bytes from S3 using `get_s3_object_bytes`.
  - Parse the text using `parse_resume_text`.
  - Upsert the `Resume` model for the candidate with `s3_url` and `raw_text`.
- [ ] `GET /me`:
  - Return candidate's active resume details (excluding full raw text for brevity unless requested).
- [ ] `DELETE /me`:
  - Soft delete or hard delete the resume record.

### 5. Integration
- [ ] Add `PyMuPDF` to `requirements.txt`.
- [ ] Register `resumes_router` in `backend/app/main.py`.

## Verification
- [ ] `POST /upload-url` returns a valid AWS S3 signed URL.
- [ ] `POST /confirm` correctly parses a mock PDF and saves `raw_text` to the database.
