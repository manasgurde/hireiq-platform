# Phase 4, Plan 2: Job Applications (HIQ-035 to HIQ-037)

**Target Phase:** 04-resume-applications-sprint-4
**Goal:** Create Application models and implement `POST /applications` with 6 pre-flight validations.

## Technical Context
- **Dependencies:** 04-01-PLAN.md (Resume model), 03-02-PLAN.md (Job model).
- **Key Constraints:** Must enforce constraints before writing to DB: Job exists, active, role candidate, has resume, no duplicate application. Status workflow uses an Enum.
- **New files:** `backend/app/models/application.py`, `backend/app/schemas/application.py`, `backend/app/api/v1/applications.py`
- **Modify:** `backend/app/main.py`

## Action Plan

### 1. Database Model (`backend/app/models/application.py`)
- [ ] Create `ApplicationStatus` Python enum: `applied`, `reviewing`, `shortlisted`, `rejected`, `hired`.
- [ ] Create `Application` SQLAlchemy model:
  - Columns: `id` (UUID), `job_id` (FK), `candidate_id` (FK), `resume_id` (FK), `status` (String/Enum), `created_at`, `updated_at`.
  - Add `UniqueConstraint("job_id", "candidate_id", name="uq_job_candidate")`.

### 2. Schemas (`backend/app/schemas/application.py`)
- [ ] Create `ApplicationCreate(BaseModel)`: `job_id: UUID`.
- [ ] Create `ApplicationResponse(BaseModel)` with all fields.

### 3. API Endpoints (`backend/app/api/v1/applications.py`)
- [ ] `POST /`:
  - Enforce `require_candidate`.
  - Pre-flight 1: Check `Job` exists.
  - Pre-flight 2: Check `Job.is_active` and `not Job.is_deleted`.
  - Pre-flight 3: (Role checked by dependency).
  - Pre-flight 4: Query `Resume` where `candidate_id == current_user.id`. Fail if none.
  - Pre-flight 5: Query `Application` for existing `(job_id, candidate_id)`. Fail if exists.
  - Insert `Application` with `status="applied"`.

### 4. Integration
- [ ] Register `applications_router` in `backend/app/main.py`.

## Verification
- [ ] `POST /applications` without a resume returns 400.
- [ ] `POST /applications` for a closed job returns 400.
- [ ] Double-applying returns 409 Conflict.
