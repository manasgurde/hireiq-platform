# Phase 4, Plan 3: Applications API (HIQ-038 to HIQ-040)

**Target Phase:** 04-resume-applications-sprint-4
**Goal:** Implement job seeker application history `GET /applications/me`, recruiter applicant list `GET /applications/job/{id}`, and `PATCH /applications/{id}/status`.

## Technical Context
- **Dependencies:** 04-02-PLAN.md (Application model and Enum)
- **Key Constraints:** Enforce recruiter ownership when viewing job applications or changing statuses.
- **Modify:** `backend/app/api/v1/applications.py`, `backend/app/schemas/application.py`

## Action Plan

### 1. Schemas (`backend/app/schemas/application.py`)
- [ ] Create `ApplicationStatusUpdate(BaseModel)`: `status: ApplicationStatus`.

### 2. API Endpoints (`backend/app/api/v1/applications.py`)
- [ ] `GET /me`:
  - Enforce `require_candidate`.
  - Return all `Application` records for `current_user.id`, joined with `Job` table to return job titles and statuses.
- [ ] `GET /job/{job_id}`:
  - Enforce `require_recruiter`.
  - Validate that `job.recruiter_id == current_user.id`.
  - Return all `Application` records for `job_id`, joined with `User` and `Profile` to show applicant details.
- [ ] `PATCH /{application_id}/status`:
  - Enforce `require_recruiter`.
  - Fetch `Application` and joined `Job`.
  - Validate `Job.recruiter_id == current_user.id`.
  - Update `Application.status` and commit.

## Verification
- [ ] Candidate can view their applied jobs.
- [ ] Recruiter can view candidates who applied to their jobs, but not to other recruiters' jobs.
- [ ] Recruiter can successfully move a candidate from `applied` to `shortlisted`.
