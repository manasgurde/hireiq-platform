# Phase 2, Plan 3: Auth Screens UI (HIQ-019 to HIQ-020)

**Target Phase:** 02-authentication-security-sprint-2
**Goal:** Implement the Next.js Login page and 2-step Registration page with inline validation, dark mode, and zxcvbn password strength meter.

## Technical Context
- **Dependencies:** 02-01-PLAN.md (auth API endpoints exist), Phase 1 frontend (Next.js 14, Tailwind v4, shadcn/ui, Zustand stores)
- **Key Constraints:** Must use App Router file-based routing. Pages must be fully functional in dark mode. Client-side forms with React Hook Form + Zod validation (no API call on validation error). Access token stored in Zustand authStore only (never localStorage). httpOnly refresh cookie is set by the backend.
- **New files:** `frontend/app/(auth)/login/page.tsx`, `frontend/app/(auth)/register/page.tsx`, `frontend/app/(auth)/layout.tsx`, `frontend/lib/api.ts`, `frontend/components/auth/PasswordStrengthMeter.tsx`
- **Modify:** `frontend/store/authStore.ts` (add login/register actions), `frontend/app/globals.css` (auth design tokens)

### 2. API Client (`frontend/lib/api.ts`)
- [ ] Create `frontend/lib/api.ts` using `axios`:
  - Base URL: `process.env.NEXT_PUBLIC_API_URL || "http://localhost:8000"`.
  - `withCredentials: true` (allows httpOnly cookies to flow with requests).
  - Axios response interceptor: on 401 → attempt `POST /v1/auth/refresh` → retry original request → if refresh also fails → clear authStore and redirect to `/login`.
- [ ] Export typed helper functions:
  - `authApi.register(data: RegisterRequest)` → `POST /v1/auth/register`
  - `authApi.login(data: LoginRequest)` → `POST /v1/auth/login`
  - `authApi.logout()` → `POST /v1/auth/logout`
  - `authApi.refreshToken()` → `POST /v1/auth/refresh`

- [ ] Create `frontend/components/auth/` directory.
- [ ] Create `PasswordStrengthMeter.tsx`:

  - Uses `zxcvbn()` to compute score (0–4).

### 5. Login Page (`frontend/app/(auth)/login/page.tsx`) (HIQ-019)
- [ ] Create the login page with Next.js `"use client"` directive.
- [ ] **Metadata:** Export `metadata = { title: "Login — HireIQ", description: "Sign in to your HireIQ account" }` in a separate server component.
- [ ] **Form schema (Zod):**
  ```typescript

    email: z.string().email("Please enter a valid email"),
  ```

- [ ] **UX:** Loading spinner on submit button during request. Button disabled during loading.
- [ ] **Dark mode:** Full dark mode styling using `dark:` Tailwind classes.

### 6. Register Page (`frontend/app/(auth)/register/page.tsx`) (HIQ-020)
- [ ] Create the 2-step registration page with `"use client"` directive.
- [ ] **Step 1 — Role Selection:**
  - Two large card options: **Job Seeker** and **Recruiter** with icon, label, and description.
  - On click, store selected role in local state, advance to Step 2.
  - Animated card border highlight on selection.
- [ ] **Step 2 — Details Form:**
  - Show "Back to role selection" button.
  - **Form schema (Zod):**
    ```typescript
    const registerSchema = z.object({
      email: z.string().email(),
      name: z.string().min(2, "Name must be at least 2 characters"),
      password: z.string()
        .min(8, "Password must be at least 8 characters")
        .regex(/[A-Z]/, "Must contain uppercase")
        .regex(/[0-9]/, "Must contain a number"),
      confirmPassword: z.string(),
    }).refine(d => d.password === d.confirmPassword, {
      message: "Passwords do not match",
      path: ["confirmPassword"],
    })
    ```
  - **Fields:** Name, Email, Password, Confirm Password.
  - **Password Strength Meter:** Render `<PasswordStrengthMeter password={watchedPassword} />` below the password field.
  - **Submit:** Call `authApi.register({ ...formData, role: selectedRole })`. On success: store tokens, redirect to `/dashboard`.
  - **API error:** Show red alert box.
- [ ] **Progress indicator:** Two-dot step indicator at the top (dot 1 = role, dot 2 = details).
- [ ] **Dark mode:** Full dark mode styling.

### 7. Zustand AuthStore Update
- [ ] Update `frontend/store/authStore.ts`:
  - Add `login(userData: User, token: string)` action → sets `user`, `accessToken`, `isAuthenticated: true`.
  - Add `logout()` action (calls `authApi.logout()` then resets state).

### 8. Route Protection (Middleware)
- [ ] Create `frontend/middleware.ts`:
  - If user visits `/dashboard` or `/recruiter` without `logged_in` cookie → redirect to `/login`.
  - If user visits `/login` or `/register` while `logged_in` cookie is present → redirect to `/dashboard`.
  - Use `request.cookies.get("logged_in")` for check.
