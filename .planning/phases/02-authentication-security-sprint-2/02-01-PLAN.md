# Phase 2, Plan 1: Auth APIs (HIQ-011 to HIQ-014)

**Target Phase:** 02-authentication-security-sprint-2
**Goal:** Implement the four core authentication endpoints — register, login (timing-attack-safe), refresh (Redis RTR), and logout.

## Technical Context
- **Dependencies:** 01-01-PLAN.md (users table exists), 01-02-PLAN.md (FastAPI app factory, Redis pool)
- **Key Constraints:** Must use PyJWT + passlib[bcrypt] (rounds=12). Refresh tokens stored as httpOnly cookies. Redis RTR with family-based reuse detection. Timing attack protection via dummy hash.
- **New files:** `backend/app/api/v1/auth.py`, `backend/app/core/security.py`, `backend/app/schemas/auth.py`, `backend/app/models/user.py`
- **Existing:** `backend/app/main.py` (register router), `backend/app/core/redis.py` (Redis pool)

## Action Plan

### 1. SQLAlchemy User ORM Model
- [ ] Create `backend/app/models/__init__.py`.
- [ ] Create `backend/app/models/user.py` defining the `User` SQLAlchemy ORM model mapped to the existing `users` table.
  - Columns: `id` (UUID), `email` (String 255, unique, indexed), `password_hash` (String 255), `role` (String 50, default `"candidate"`), `created_at`, `updated_at`.
  - Add a `__tablename__ = "users"` declaration.
- [ ] Create `backend/app/core/database.py` defining the async SQLAlchemy engine and `AsyncSession` factory using `settings.DATABASE_URL`.
  - Export `get_db` FastAPI dependency (async generator yielding `AsyncSession`).

### 2. Security Utilities (`backend/app/core/security.py`)
- [ ] Configure `CryptContext` with `bcrypt` scheme and `bcrypt__rounds=12`.
- [ ] Compute and store `DUMMY_HASH` at module load time (for timing attack protection).
- [ ] Implement `hash_password(password: str) -> str`.
- [ ] Implement `verify_password(plain: str, hashed: str) -> bool`.
- [ ] Implement `create_access_token(data: dict, expires_delta: timedelta | None = None) -> str` (uses PyJWT, encodes `type="access"`, exp=15 min).
- [ ] Implement `create_refresh_token(user_id: str, family_id: str | None = None) -> tuple[str, str, str]` returning `(token, jti, family_id)`. RT has `type="refresh"`, exp=7d.
- [ ] Implement `decode_token(token: str) -> dict` raising `HTTPException 401` on invalid/expired tokens.

### 3. Pydantic Auth Schemas (`backend/app/schemas/auth.py`)
- [ ] Create `backend/app/schemas/__init__.py`.
- [ ] Define `RegisterRequest(email: EmailStr, password: str, role: Literal["candidate","recruiter"])`.
  - Password validator: min 8 chars, at least 1 uppercase, 1 lowercase, 1 digit.
- [ ] Define `LoginRequest(email: EmailStr, password: str)`.
- [ ] Define `TokenResponse(access_token: str, token_type: str = "bearer")`.
- [ ] Define `UserOut(id: UUID, email: str, role: str, created_at: datetime)`.

### 4. Redis RTR Helpers (`backend/app/core/redis.py` — extend)
- [ ] Add `store_refresh_token(jti: str, user_id: str, family_id: str)` → `SETEX rt:{jti} 604800 {...}` + `SADD rt_family:{family_id} {jti}`.
- [ ] Add `rotate_refresh_token(old_jti: str, new_jti: str, user_id: str, family_id: str)` → atomic pipeline: `DEL rt:{old_jti}`, `SETEX revoked_rt:{old_jti} 60 reused`, `SETEX rt:{new_jti}`, `SADD rt_family:{family_id} {new_jti}`, `SREM rt_family:{family_id} {old_jti}`, `EXPIRE rt_family:{family_id} 604800`.
- [ ] Add `check_refresh_reuse(jti: str) -> bool` → check `EXISTS revoked_rt:{jti}`.
- [ ] Add `get_refresh_payload(jti: str) -> dict | None` → `GET rt:{jti}` and JSON parse.
- [ ] Add `invalidate_token_family(family_id: str)` → `SMEMBERS rt_family:{family_id}`, delete all `rt:{member}`, then `DEL rt_family:{family_id}`.
- [ ] Add `delete_refresh_token(jti: str)` → `DEL rt:{jti}`.

### 5. Auth Router (`backend/app/api/v1/auth.py`)
- [ ] Create router with `prefix="/auth"`, `tags=["Authentication"]`.

#### POST /auth/register (HIQ-011)
- [ ] Accept `RegisterRequest` body.
- [ ] Check for existing email: if found → raise `HTTPException(409, "Email already registered")`.
- [ ] Hash password with `hash_password()`.
- [ ] Insert `User` record via async session. Commit and refresh.
- [ ] Create access token and refresh token (new family_id via `uuid4()`).
- [ ] Store refresh token in Redis via `store_refresh_token()`.
- [ ] Set `refresh_token` httpOnly cookie (Secure=True, SameSite=strict, path="/v1/auth/refresh", max_age=604800).
- [ ] Return `TokenResponse` with access token + `UserOut`.

#### POST /auth/login (HIQ-012)
- [ ] Accept `LoginRequest` body.
- [ ] Check rate limit (dependency from plan 02-02 — the auth route is decorated but the limiter is implemented there; add a TODO comment for now).
- [ ] Fetch user by email from DB.
- [ ] If user not found: call `verify_password(plain, DUMMY_HASH)` to prevent timing attack, then raise `HTTPException(401, "Invalid credentials")`.
- [ ] If user found but password wrong: raise `HTTPException(401, "Invalid credentials")`.
- [ ] Create access + refresh tokens. Store RT in Redis.
- [ ] Set httpOnly cookie.
- [ ] Return `TokenResponse` + `UserOut`.

#### POST /auth/refresh (HIQ-013)
- [ ] Read `refresh_token` from cookie.
- [ ] Decode the RT using `decode_token()`.
- [ ] Assert `payload["type"] == "refresh"`.
- [ ] Extract `jti` and `family_id` from payload.
- [ ] Check reuse: if `check_refresh_reuse(jti)` → call `invalidate_token_family(family_id)` → raise `HTTPException(401, "Refresh token reused — session invalidated")`.
- [ ] Check active: if `get_refresh_payload(jti)` returns None → raise `HTTPException(401, "Refresh token not found or expired")`.
- [ ] Rotate: create new RT, call `rotate_refresh_token(old_jti, new_jti, user_id, family_id)`.
- [ ] Create new access token.
- [ ] Set new httpOnly cookie.
- [ ] Return `TokenResponse`.

#### POST /auth/logout (HIQ-014)
- [ ] Read `refresh_token` cookie.
- [ ] Decode RT, extract `jti`.
- [ ] Call `delete_refresh_token(jti)`.
- [ ] Clear the `refresh_token` cookie (max_age=0).
- [ ] Return `{"message": "Logged out successfully"}`.

### 6. Register Router in main.py
- [ ] Import `auth_router` from `app.api.v1.auth`.
- [ ] Replace the commented-out placeholder with `app.include_router(auth_router, prefix="/v1")`.

## Verification
- [ ] `POST /v1/auth/register` with valid data → 201 with access token and httpOnly cookie.
- [ ] `POST /v1/auth/register` with duplicate email → 409.
- [ ] `POST /v1/auth/login` with wrong password → 401 (response time ~200ms for both found/not-found users).
- [ ] `POST /v1/auth/refresh` with valid cookie → 200 new access token, old cookie rotated.
- [ ] `POST /v1/auth/refresh` with reused old token → 401, family invalidated.
- [ ] `POST /v1/auth/logout` → 200, cookie cleared.
