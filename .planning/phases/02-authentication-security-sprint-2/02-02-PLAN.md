# Phase 2, Plan 2: Security Middlewares (HIQ-015 to HIQ-018)

**Target Phase:** 02-authentication-security-sprint-2
**Goal:** Build JWT validation middleware (`get_current_user`), RBAC `require_role` dependency factory, Redis sliding window rate limiter, and a global JSON error envelope.

## Technical Context
- **Dependencies:** 02-01-PLAN.md (security.py, User ORM model, Redis helpers exist)
- **Key Constraints:** Rate limiter must be reusable across routes via FastAPI Depends. RBAC checks must short-circuit with 403 before any DB queries on protected routes. Error envelope must cover validation, HTTP, and unhandled exceptions without leaking stack traces.
- **New files:** `backend/app/middlewares/rate_limiter.py`, `backend/app/core/deps.py`
- **Modify:** `backend/app/main.py` (register exception handlers)

## Action Plan

### 1. JWT Auth Dependency (`backend/app/core/deps.py`)

#### `get_current_user` (HIQ-015)
- [ ] Create `backend/app/core/deps.py`.
- [ ] Import `OAuth2PasswordBearer` and configure with `tokenUrl="/v1/auth/login"`.
- [ ] Implement `async def get_current_user(token: str = Depends(oauth2_scheme), db: AsyncSession = Depends(get_db)) -> User`:
  - Decode the token with `decode_token()`. On `JWTError` → raise `HTTPException(401, "Could not validate credentials")`.
  - Assert `payload["type"] == "access"`. If not → raise 401.
  - Extract `user_id = payload["sub"]`.
  - Query the DB for the user by ID. If not found → raise 401.
  - Return the `User` ORM instance.
- [ ] Implement `async def get_current_active_user(current_user: User = Depends(get_current_user)) -> User`:
  - Can be extended later with is_active check.
  - Returns the user directly.

#### RBAC Dependency Factory (HIQ-016)
- [ ] Define `class RoleChecker`:
  - `__init__(self, allowed_roles: list[str])`.
  - `__call__(self, current_user: User = Depends(get_current_user)) -> User` — checks if `current_user.role` is in `allowed_roles`. If not → raise `HTTPException(403, "Insufficient permissions")`.
  - Returns `current_user` on success.
- [ ] Create pre-built shortcuts:
  - `require_recruiter = RoleChecker(["recruiter", "admin"])`
  - `require_candidate = RoleChecker(["candidate", "admin"])`
  - `require_admin = RoleChecker(["admin"])`
- [ ] Add ownership check helper: `async def verify_resource_owner(resource_owner_id: UUID, current_user: User) -> None`:
  - If `resource_owner_id != current_user.id` and `current_user.role != "admin"` → raise `HTTPException(403, "You don't own this resource")`.

### 2. Redis Sliding Window Rate Limiter (`backend/app/middlewares/rate_limiter.py`) (HIQ-017)
- [ ] Create `backend/app/middlewares/rate_limiter.py`.
- [ ] Write the Lua script string `RATE_LIMIT_LUA` for sliding window logic:
  - `ZREMRANGEBYSCORE key -inf (now - window_ms)` → remove expired entries
  - `ZCARD key` → count entries in window
  - If count < limit → `ZADD key now (now + random float suffix)` + `EXPIRE` → return 1
  - Else → return 0
- [ ] Implement `class SlidingWindowRateLimiter`:
  - `__init__(self, requests: int = 10, window_seconds: int = 60)`.
  - `async def __call__(self, request: Request)` → FastAPI dependency.
    - Build key: `rate:{request.url.path}:{client_host}`.
    - Execute `redis_client.eval(RATE_LIMIT_LUA, 1, key, now_ms, window_ms, requests)`.
    - If result is 0 → raise `HTTPException(429, detail="Too many requests. Please slow down.")`, include `Retry-After` header.
- [ ] Create pre-built limiters:
  - `auth_rate_limit = Depends(SlidingWindowRateLimiter(requests=10, window_seconds=60))` — for auth endpoints
  - `api_rate_limit = Depends(SlidingWindowRateLimiter(requests=100, window_seconds=60))` — for general API
- [ ] Update `backend/app/api/v1/auth.py` to add `dependencies=[auth_rate_limit]` to the router or to individual endpoints (register, login, refresh).

### 3. Global Error Envelope (`backend/app/main.py` update) (HIQ-018)
- [ ] Import `RequestValidationError` from `fastapi.exceptions` and `HTTPException` from `fastapi`.
- [ ] Import `Exception` handler from `starlette`.
- [ ] Register `@app.exception_handler(RequestValidationError)`:
  - Build `details` list from `exc.errors()`: extract field path (join `e["loc"][1:]`) and issue (`e["msg"]`).
  - Return `JSONResponse(status_code=422, content={"success": False, "error": {"code": "VALIDATION_ERROR", "message": "Input validation failed.", "details": details}})`.
- [ ] Register `@app.exception_handler(HTTPException)`:
  - Return `JSONResponse(status_code=exc.status_code, content={"success": False, "error": {"code": exc.detail if isinstance(exc.detail, str) else "HTTP_ERROR", "message": str(exc.detail)}})`.
- [ ] Register `@app.exception_handler(Exception)`:
  - Log with `logger.error("unhandled_exception", error=str(exc), exc_info=True)`.
  - Return `JSONResponse(status_code=500, content={"success": False, "error": {"code": "INTERNAL_SERVER_ERROR", "message": "An unexpected error occurred."}})`.
- [ ] Ensure the `/v1/health` endpoint returns the envelope-compliant success shape: `{"success": True, "data": {"status": "ok", "version": "..."}}`.
