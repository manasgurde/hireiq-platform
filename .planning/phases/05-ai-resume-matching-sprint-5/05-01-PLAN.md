# Phase 5, Plan 1: AI Service & spaCy (HIQ-041, HIQ-042, HIQ-046)

**Target Phase:** 05-ai-resume-matching-sprint-5
**Goal:** Setup the FastAPI `ai_service` project, load spaCy models using lifespan events, and implement the NER skills/experience parsing engine.

## Technical Context
- **Dependencies:** 05-RESEARCH.md (spaCy NER usage and FastAPI lifespan pattern).
- **New Directory:** `ai_service/` (will run on port 8001, distinct from `backend/` running on 8000).
- **New files:** `ai_service/main.py`, `ai_service/requirements.txt`, `ai_service/app/schemas.py`, `ai_service/app/services/parser.py`

## Action Plan

### 1. Initialize `ai_service`
- [ ] Create `ai_service/requirements.txt` with `fastapi`, `uvicorn`, `spacy`, `scikit-learn`, `sentence-transformers`, `pydantic`.
- [ ] Create `ai_service/main.py` scaffolding with FastAPI and `@asynccontextmanager` `lifespan` event.
- [ ] In the lifespan event, configure `spacy.load("en_core_web_sm")` with a custom `EntityRuler` containing common tech skills (Python, React, FastAPI, SQL, etc.), storing it in `app.state.nlp`.

### 2. NER Parsing Engine (`ai_service/app/services/parser.py`)
- [ ] Create `def parse_resume_text(text: str, nlp_model) -> dict`.
- [ ] Implement logic to run `nlp_model(text)` and extract entities:
  - `ORG` -> past employers/schools.
  - `DATE` -> tenures.
  - `SKILL` -> core competencies.
- [ ] Deduplicate extracted lists.

### 3. Parse API Endpoint
- [ ] Create `POST /parse` endpoint in `ai_service/main.py`.
- [ ] Accepts `{"text": "resume raw text"}`.
- [ ] Returns `{ "organizations": [...], "dates": [...], "skills": [...] }`.

## Verification
- [ ] The `ai_service` starts up without errors and successfully preloads the spaCy model.
- [ ] `POST /parse` correctly extracts skills mapped by the `EntityRuler`.
