# Phase 1, Plan 3: Frontend & CI Scaffolding

**Target Phase:** 01-foundation-project-setup-sprint-1
**Goal:** Initialize the Next.js 14 frontend project with Tailwind CSS, configure the Design System tokens, set up Zustand stores, and configure the GitHub Actions CI pipeline.

## Technical Context
- **Dependencies:** None
- **Key Constraints:** Must strictly follow the color, typography, spacing, and shadow tokens from FSD Section 1. Must use Next.js 14 App Router.

## Action Plan

### 1. Next.js 14 Project Setup
- [ ] Initialize the Next.js project in `frontend/` using `npx create-next-app@latest` with TypeScript, ESLint, Tailwind CSS, and App Router enabled.
- [ ] Initialize `shadcn/ui` with the default styling.

### 2. Design System & Tailwind Configuration
- [ ] Update `tailwind.config.ts` to include the specific color palette (Primary Blue, Indigo, Teal, Slate Dark, etc.) from FSD Section 1.1.
- [ ] Update typography settings in `tailwind.config.ts` (Display XL, Heading 1, etc.) per FSD Section 1.2.
- [ ] Configure the custom shadow and border tokens in `tailwind.config.ts` per FSD Section 1.4.
- [ ] Implement dark mode toggle logic (`darkMode: 'class'`) and apply background base colors to `app/globals.css`.

### 3. Client State (Zustand)
- [ ] Create `frontend/store/authStore.ts` to manage user authentication state (`user`, `accessToken`, `isAuthenticated`).
- [ ] Create `frontend/store/uiStore.ts` to manage UI preferences (`darkMode`, `sidebarOpen`).

### 4. GitHub Actions CI Pipeline
- [ ] Create `.github/workflows/ci.yml` in the project root.
- [ ] Add jobs to run on `push` and `pull_request` to `main`.
- [ ] Configure `backend-lint` step using `ruff`.
- [ ] Configure `frontend-lint` step using `eslint`.
- [ ] Configure `backend-test` step using `pytest` with coverage thresholds.
- [ ] Configure `frontend-test` step using `jest` with coverage thresholds.
- [ ] Configure Docker image build steps for both backend and frontend to verify they build correctly.

## Verification
- [ ] Run `npx next dev` inside `frontend/` and verify the app starts on port 3000 without TypeScript errors.
- [ ] Verify `tailwind.config.ts` strictly matches the FSD design tokens.
- [ ] Verify GitHub Actions configuration syntax using a linter or manual review.
