# Phase 1, Plan 1: Project Scaffolding & Database

**Target Phase:** 01-foundation-project-setup-sprint-1
**Goal:** Setup the monorepo root structure, Docker Compose configuration, and PostgreSQL database schemas via Alembic migrations.

## Technical Context
- **Dependencies:** None
- **Key Constraints:** Must match the FSD and BSD structures. The databases must be reachable from host for local development.

## Action Plan

### 1. Root Monorepo Structure & Git
- [ ] Create `backend/`, `frontend/`, and `ai_service/` directories.
- [ ] Create `.gitignore` to ignore `.env`, `__pycache__`, `node_modules`, `.next`, and `.agent`.
- [ ] Create `README.md` with startup instructions.

### 2. Docker Compose Configuration
- [ ] Create `docker-compose.yml` at the project root.
- [ ] Define the `db` service using `postgres:15-alpine`.
- [ ] Define the `redis` service using `redis:7-alpine`.
- [ ] Define placeholders for `api`, `ai`, and `frontend` services with build contexts.

### 3. Backend PostgreSQL Migrations (Alembic)
- [ ] Initialize Alembic in the `backend/` directory (`alembic init alembic`).
- [ ] Configure `alembic.ini` for async URL support.
- [ ] Create the initial migration file for creating the 5 core tables (`users`, `jobs`, `resumes`, `applications`, `interview_sessions`).
- [ ] Include GIN indexes for full-text search in jobs, and B-tree indexes for foreign keys.
- [ ] Ensure the UNIQUE constraint `(user_id, job_id)` on the `applications` table is present.

## Verification
- [ ] Verify `docker-compose config` runs without errors.
- [ ] Verify Alembic downgrade and upgrade scripts run successfully against the local database.
