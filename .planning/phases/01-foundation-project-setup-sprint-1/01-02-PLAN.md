# Phase 1, Plan 2: Core API Shell

**Target Phase:** 01-foundation-project-setup-sprint-1
**Goal:** Configure the FastAPI application factory, register routers, configure Redis pool, configure Pydantic BaseSettings, and inject JSON logging middleware.

## Technical Context
- **Dependencies:** 01-01-PLAN.md (Database setup and Docker environment)
- **Key Constraints:** Must use `structlog` for logging and implement a unique request ID. Settings must be loaded securely via Pydantic `BaseSettings`.

## Action Plan

### 1. Pydantic Settings & Redis Pool
- [ ] Create `backend/app/core/config.py` using `BaseSettings` to parse environment variables (DATABASE_URL, REDIS_URL, JWT secrets, AWS keys).
- [ ] Create `backend/app/core/redis.py` utilizing `redis.asyncio` to create an async connection pool.
- [ ] Write helper functions for Redis cache invalidation (`delete_pattern`) and TTL fetching (`cache_or_fetch`).

### 2. Logging Middleware
- [ ] Create `backend/app/middlewares/logging_middleware.py`.
- [ ] Implement a `BaseHTTPMiddleware` that assigns a `uuid4()` request ID.
- [ ] Inject request ID into response headers as `X-Request-ID`.
- [ ] Use `structlog` to log the method, path, status_code, and processing duration for each request.

### 3. FastAPI Application Factory
- [ ] Create `backend/app/main.py`.
- [ ] Implement the async `lifespan` context manager to initialize the DB connection and Redis pool.
- [ ] Instantiate the FastAPI app.
- [ ] Register CORS middleware allowing frontend origin (`localhost:3000`).
- [ ] Add the logging middleware.
- [ ] Register placeholder routers for `/v1/auth`, `/v1/users`, `/v1/jobs`, `/v1/resumes`, `/v1/applications`, `/v1/ai`, and `/v1/recruiter`.
- [ ] Implement a basic GET `/health` endpoint returning `{"status": "ok", "version": "1.0.0"}`.

## Verification
- [ ] Run FastAPI locally `uvicorn app.main:app --reload` and verify `/health` returns 200 OK.
- [ ] Verify `structlog` outputs JSON logs with request IDs in the console.
